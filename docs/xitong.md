# 一、系统目标

你要的不是单个写作 Agent，而是一套 **内容生产流水线系统**。

## 核心能力

系统需要支持：

1. **普通任务模式**

   * 手动输入主题
   * 联网搜索
   * 按自定义文章框架生成文章
   * 自动审核
   * 自动生成封面/配图
   * 自动推送公众号草稿箱

2. **热点自动模式**

   * 前端可配置“是否自动走热点”
   * 可配置热点类别：

     * 科技
     * 财经
     * 军事
     * 社会
     * AI
     * 消费
     * 国际
     * 行业垂类
   * 开启后系统会：

     * 自动抓取热点
     * 聚合热点
     * 筛选适合账号定位的选题
     * 自动进入内容生产链

3. **审核与风控**

   * 敏感内容过滤
   * 极限词/夸张承诺过滤
   * 行业禁用表达过滤
   * 图片内容合规过滤
   * 发布前支持人工兜底

4. **文章和图片协同**

   * 图片不是独立附件
   * 要根据文章内容、段落重点、封面定位自动生成

# 二、最适合的 AI 范式

你的需求最适合：

> **Workflow / Harness 主导 + 专职 Agent 节点协作**

不是单一万能 Agent。

## 原因

因为你的场景有两类特征：

### 1. 固定流程很强

例如：

* 热点抓取
* 热点筛选
* 搜索与资料整理
* 提纲生成
* 正文生成
* 审核
* 图片生成
* 草稿箱推送

这类步骤顺序明确，适合 **Workflow / Harness**

### 2. 局部决策很强

例如：

* 热点值不值得写
* 文章角度怎么定
* 用什么框架更合适
* 这篇文章要几张图、图该怎么画
* 某段内容是否敏感、如何改写

这类环节适合 **专职 Agent**

## 最终架构范式

所以建议采用：

> **Orchestrator + Shared State + Specialist Agents + Tools + Human Gate**

也就是：

* **工作流系统**控制流程
* **Planner Agent**定选题和提纲
* **Writer Agent**写稿
* **Reviewer Agent**审核
* **Image Agent**规划图片与生成提示词
* **Publisher Node**推送草稿箱
* **Hotspot Aggregator**在前置链路负责热点捕获

# 五、热点捕获与聚合模块设计

这是你新增的重点。

---

## 1. 热点模块目标

前端配置了“自动走热点”后，系统自动完成：

* 热点抓取
* 热点分类
* 热点聚合
* 热点去重
* 热点评分
* 热点转选题
* 进入写作链

---

## 2. 前端配置项设计

建议前端提供以下配置：

### 基础配置

* `auto_hotspot_enabled`：是否开启自动热点
* `hotspot_fetch_interval`：抓取频率
* `max_topics_per_run`：每次最多生成几篇
* `auto_publish_to_draft`：是否自动入草稿箱

### 热点分类配置

* `hotspot_categories`：

  * 科技
  * 财经
  * 军事
  * AI
  * 社会
  * 国际
  * 消费
  * 行业垂类

### 热点筛选配置

* `min_hot_score`
* `keyword_whitelist`
* `keyword_blacklist`
* `account_fit_tags`
* `duplicate_window_hours`

### 选题偏好配置

* 更偏资讯型 / 解读型 / 评论型 / 科普型
* 是否优先写高时效热点
* 是否允许争议性热点
* 是否允许敏感类主题进入候选池

---

## 3. 热点数据来源设计

热点来源可以设计成可插拔 Source Connector：

```text
Hotspot Source Adapters
  ├─ 新闻源
  ├─ 搜索趋势源
  ├─ 行业资讯源
  ├─ 社交平台趋势源
  ├─ RSS源
  └─ 自定义采集源
```

每个源统一输出结构：

```json
{
  "source": "xxx",
  "title": "xxx",
  "summary": "xxx",
  "url": "xxx",
  "publish_time": "xxx",
  "category": "科技",
  "keywords": ["AI", "芯片", "模型"],
  "heat_score_raw": 83
}
```

---

## 4. 热点聚合流程

### 第一步：抓取

从多个源抓取热点数据

### 第二步：清洗

清洗无效数据：

* 去广告
* 去垃圾内容
* 去极短无效标题

### 第三步：分类

统一分类映射：

* 科技
* 财经
* 军事
* AI
* 社会
* 国际
* 汽车
* 行业

### 第四步：聚合

同一热点多来源合并：

* 相似标题归并
* 关键词归并
* 事件聚类

### 第五步：去重

用标题相似度 + 关键词相似度 + 时间窗去重

### 第六步：评分

给热点打综合分：

```text
综合分 = 热度分 + 时效分 + 多源确认分 + 账号匹配分 - 风险分
```

### 第七步：选题化

把“热点事件”转成“可写文章题目”

例如：

热点原始事件：

* 某科技公司发布新模型

选题化后：

* 这次新模型升级，普通用户能得到什么？
* 从这次模型发布看，AI行业下一步会怎么走？
* 这次产品更新对企业侧意味着什么？

---

## 5. 热点模块输出

热点模块最终不直接输出文章，而是输出 **候选选题池**：

```json
{
  "run_id": "xxx",

      "hot_score": 91,
      "account_fit_score": 88,
      "risk_score": 12,
      "recommended": true
    }
  ]
}
```

# 六、Agent 体系设计

建议你用 4 个核心 Agent。

---

## 1. Planner Agent

### 职责

* 从主题或热点中确定写作角度
* 生成文章提纲
* 选择适合的文章框架
* 生成图片需求草案

### 输入

* 主题
* 热点摘要
* 账号定位
* 模板库
* 目标读者
* 风险等级

### 输出

* 标题候选
* 文章大纲
* 写作意图
* 图片规划草案

### 示例输出

```json
{
  "title_candidates": ["...", "..."],
  "framework": "资讯解读型",
  "outline": [
    {"section": "导语", "goal": "引出热点"},
    {"section": "事件背景", "goal": "交代事实"},
    {"section": "核心解读", "goal": "解释影响"},
    {"section": "总结", "goal": "输出观点"}
  ],
  "image_plan_seed": {
    "cover_needed": true,
    "inline_count": 2
  }
}
```

---

## 2. Writer Agent

### 职责

* 按大纲写正文
* 生成标题、导语、摘要、结尾
* 保持品牌语气
* 结合搜索事实

### 输入

* 搜索资料
* 提纲
* 模板
* 品牌语气
* 禁用表达

### 输出

* 正文初稿
* 标题候选
* 摘要
* 封面文案建议

---

## 3. Reviewer Agent

### 职责

* 检查敏感内容
* 检查夸张表述
* 检查逻辑问题
* 检查事实引用
* 检查结构完整性
* 审核失败时给出改写建议

### 审核维度

* 政治敏感
* 暴力/色情/违法
* 医疗功效误导
* 金融投资承诺
* 极限词
* 不实引导
* 标题党
* 品牌违禁表述
* 图片文案冲突

### 输出

```json
{
  "passed": false,
  "risk_level": "medium",
  "issues": [
    {"type": "extreme_word", "content": "最强", "fix": "建议改为“较受关注”"},
    {"type": "unsupported_claim", "content": "一定会涨", "fix": "改为“存在上涨预期”"}
  ],
  "rewrite_required": true
}
```

---

## 4. Image Agent

这是你的重点。

### 角色定位

不是单纯出图工具，而是一个 **理解文章内容的专职 Agent**

---

### 4.1 Image Planner

负责决定：

* 需要几张图
* 哪张是封面图
* 哪些段落需要配图
* 每张图的用途是什么

### 4.2 Image Prompt Composer

负责把文章内容转成图像需求：

* 主体对象
* 场景
* 视觉重点
* 色调
* 构图
* 风格
* 是否留白
* 是否偏纪实/科技感/财经感/军事风格

### 4.3 Image Reviewer

负责判断：

* 图是否符合文章主题
* 是否有违规元素
* 是否适合公众号封面
* 是否保留标题区
* 是否过于花哨影响可读性

### Image Agent 输入

* 标题
* 摘要
* 提纲
* 段落重点
* 品牌视觉规范
* 图片风格配置

### Image Agent 输出

```json
{
  "cover": {
    "prompt": "xxx",
    "purpose": "封面吸引点击",
    "style": "科技未来感"
  },
  "inline_images": [
    {
      "section": "核心解读",
      "prompt": "xxx",
      "purpose": "解释关键概念"
    }
  ]
}
```

# 十四、任务执行策略

---

## 1. 自动热点任务执行策略

定时触发：

* 每小时抓一次热点
* 每次聚合分类
* 每类按分数选 Top N
* 每次任务最多生成 M 篇

## 2. 去重策略

避免重复写同一热点：

* 24 小时去重
* 同 cluster_id 不重复生成
* 标题相似度阈值控制

## 3. 风险控制策略

高风险热点不自动进入内容链：

* 军事/财经等高敏类可单独提高审核等级
* 敏感热点只能入候选池，需人工批准后写作
