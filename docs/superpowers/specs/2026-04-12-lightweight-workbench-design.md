# 轻量工作台与列表化收敛设计

## 1. 背景与目标

在完成全站白天 / 黑夜切换后，当前前端外观仍然存在几个明显问题：

- 浅色模式下仍有部分模块残留偏深色的壳层和背景
- 左侧菜单虽然已经主题化，但整体尺寸和页面比例仍不够稳定
- 页面顶部 Hero 区偏大，标题和描述占用了过多垂直空间
- 多个业务页仍然依赖较强的卡片式包装，视觉上偏“展示型工作台”，不够简洁

本次改造的目标不是继续强化品牌感，而是把当前界面收敛成更克制的轻量工作台：

- 风格更简洁，减少装饰性大块面板
- 全站浅色模式完整成立，不再出现明显深色残留
- 左侧菜单桌面端固定宽度，并统一菜单项尺寸
- 页面顶部区域压缩为紧凑页头
- 各页面尽量使用列表、表格、分组表单和工具栏
- 尽量减少卡片，仅保留必要的轻量容器

## 2. 设计结论

### 2.1 总体方向

采用“轻量工作台 + 列表优先”的后台风格，而不是继续沿用当前较强的品牌展示型布局。

最终视觉应接近：

- 更稳定的导航骨架
- 更紧凑的页面头部
- 更平的容器层级
- 更高比例的列表 / 表格 / 分组表单

### 2.2 菜单策略

左侧菜单桌面端固定宽度为 `216px`。

同时统一以下尺寸：

- 菜单图标容器尺寸
- 菜单项最小高度
- 菜单项内边距
- 菜单选中态与 hover 态的边框 / 背景强度

目标是让所有页面进入后，内容区宽度和左栏比例都稳定一致。

### 2.3 顶部区域策略

当前 Hero 区降级为“紧凑页头”，只保留：

- 页面标题
- 一句简短说明
- 页面级主要操作

不再保留：

- 大面积装饰性渐变背景
- 过大的标题字号
- 占据首屏高度的描述区
- 非必要的指标卡

### 2.4 容器策略

卡片不再作为默认页面容器。

新的优先级调整为：

1. 工具栏
2. 列表 / 表格
3. 分组表单
4. 轻量信息块
5. 少量必要卡片

卡片仅在以下场景保留：

- 需要明确隔离的配置分组
- 需要强调的摘要信息
- 预览区域

## 3. 全局布局设计

### 3.1 工作台骨架

全局保留当前左侧导航 + 主内容区的骨架，但做以下收敛：

- 左侧导航固定宽度 `216px`
- 主内容区 padding 适当缩小
- 顶部工具条与页面头合并为更紧凑的一层
- 内容画布减少额外包裹层

### 3.2 左侧菜单

左侧菜单的设计目标是“稳定、安静、清楚”，而不是成为视觉主角。

具体要求：

- 固定宽度 `216px`
- 菜单项高度统一
- 图标区统一尺寸
- 激活态更轻，避免过重渐变和过强阴影
- 浅色模式下仍保持清晰的对比，不出现深色残留

### 3.3 页面头

页面头统一改为紧凑结构：

- 标题字号明显小于当前 Hero 标题
- 描述限制为一到两行
- 描述使用次级文本色
- 与列表工具栏之间的间距缩小

若页面不需要描述，可直接只显示标题和工具栏。

## 4. 页面级改造策略

### 4.1 TaskCreate

改为紧凑创建页：

- 页面头缩小
- 主区域优先使用分组表单
- 辅助说明区改为轻量列表，而不是大卡片展示

### 4.2 TaskDetail

改为“摘要 + 分段列表”：

- 顶部仅保留任务标题、状态和关键动作
- 中部将轨道、产物、日志等内容尽量收敛成分段列表
- 减少强调性的重背景壳层

### 4.3 History

优先改为列表页：

- 以列表或表格作为主视图
- 过滤、排序、分页集中在顶部工具栏
- 大卡片历史视图弱化

### 4.4 ArticleManage

改为“列表 + 预览”结构：

- 主体为文章列表或表格
- 预览保留，但弱化容器装饰
- 主题、推送、状态等操作集中到工具栏或行操作中

### 4.5 ScheduleManage

改为标准后台页：

- 筛选区
- 列表区
- 编辑动作使用弹窗 / 抽屉 / 表单分组

减少页面上的多层容器包裹。

### 4.6 StyleConfig

保持分组配置结构，但弱化卡片感：

- 分组之间用轻边框或间距区分
- 预览区保留，但不使用厚重展示型容器

### 4.7 ModelConfig / AccountConfig

以配置分组 + 列表为主：

- 列表管理项
- 分组表单编辑
- 减少装饰性块面

## 5. 浅色模式收敛要求

这次改造必须继续清理浅色模式遗漏项。

重点包括：

- 页面壳层背景
- 顶部页头背景
- 左侧菜单 hover / active 态
- 业务页内部容器背景
- 预览框周边容器
- 列表行、工具栏、空状态、辅助说明文本

要求是“浅色模式完整成立”，而不是仅靠全局背景变量变白。

## 6. 样式系统约束

### 6.1 样式来源

优先使用：

- Ant Design token
- `--app-*` 语义变量

避免继续新增：

- 直接写死的深色值
- 展示型大渐变
- 无明确语义的临时颜色常量

### 6.2 视觉强度

整体视觉需要降噪：

- 阴影减轻
- 渐变减少
- 圆角适中
- 文案层级更清楚
- 组件边界更简洁

### 6.3 列表优先

页面内容尽量以下列结构组织：

- 页面头
- 工具栏
- 列表 / 表格
- 分组表单
- 轻量辅助说明

避免大量“卡片套卡片”的组织方式。

## 7. 范围与非目标

### 7.1 本次包含

- 全局外壳收敛
- 左侧菜单固定尺寸
- 顶部页头压缩
- 浅色模式残留样式清理
- 各主页面结构向列表 / 表格 / 分组表单收敛
- 减少现有卡片数量和视觉厚度

### 7.2 本次不包含

- 业务逻辑重构
- API 层改造
- 页面功能增删
- 全新的品牌视觉表达
- 大规模动画设计

## 8. 验收标准

- 桌面端左侧菜单固定宽度为 `216px`
- 菜单项高度、图标区尺寸、内边距稳定统一
- 每个页面顶部区域明显比当前更紧凑
- 浅色模式下不再出现明显深色残留壳层
- 各页面主体以列表 / 表格 / 分组表单为主
- 大卡片数量明显减少
- 前端测试继续通过，必要测试同步更新
